MODUL 1

2. Operater i kontrolor — Modul 1 Linija

2.1 Prijava

  1. Otvorite aplikaciju u Chrome ili Edge.
  2. Unesite email i lozinku (od admina).
  3. Greške: pogrešna lozinka · nije u radnici · deaktiviran → admin.
Operator i kontrolor uvek rade u Modulu 1. Modul 2 ne koriste.

2.2 Atributivne — dnevni rad

KorakŠta radite
A — ID delaSken barkod ili unos (npr. NT-001). Proverite naziv i crtež. RN ako traži.
B — Poka-yokeČekirajte SVE stavke. Bez toga — ne možete dalje.
C — UnosOK ili NOK. Za NOK: greška, zona vozila, foto.
D — SnimiKontrolor: LISTA / SNIMI u bazu.

2.3 Merljive — dnevni rad

KorakŠta radite
ID / serijaID dela (npr. NM-001), pogon A/B, RN.
Poka-yokeID, RN, crtež, instrument. Kalibracija mora biti OK.
UnosRučno (tastatura) ili digitalno (USB merilo). LSL/USL → NOK van granice.

2.4 Tabovi (Modul 1)

UlogaAtributivneMerljive
Operatorsamo UNOSsamo UNOS
KontrolorUNOS + LOGUNOS + LOG

2.5 Pravila

  1. Ne preskačite poka-yoke.
  2. Tačan ID dela — pogrešan ID = pogrešne granice.
  3. Ne delite lozinku.
  4. Wi‑Fi prekid — unos ide u red, sinhronizuje se posle.
  5. Kalibracija istekla = blokada digitalnog unosa.
Operater ne koristi SPC karte. Vaš posao je tačan unos na liniji. Karte radi inženjer u Modulu 2.

MODUL 2

3. Inženjer kvaliteta — Modul 2 Analitika

3.1 Ulaz

  1. Prijava → početni ekran → Modul 2 — Analitika.
  2. Izaberite Atributivne ili Merljive.
Prebacivanje Modul 1 ↔ 2 samo na početnom ekranu.

3.2 Šta je analitika?

Sve posle unosa sa linije:

  • Stabilan proces? → SPC karte
  • Sposoban proces? → Cp/Cpk
  • Koje greške dominiraju? → Pareto
  • Trend? → RTY, DPMO, dashboard
  • Trag? → Trasabilitet
  • Rešenje? → 8D, eskalacije, STANJE
